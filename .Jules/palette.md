## 2025-02-23 - Add explicit accessibility labels to Setup and Strategy Note forms
**Learning:** Found that custom UI elements (like strategy note input lists and season select dropdowns) were missing basic accessibility hooks (`id` and `htmlFor` linking) and `aria-label` definitions in interactive inputs. While the visual text cues exist (like placeholder text or adjacent labels), standard HTML semantics required explicit hooks to correctly expose their relationship to screen readers. React's `useId` is optimal for component reuse, but static IDs work when it's single-instance.
**Action:** When adding inputs or building forms with custom labels, always tie them together with `htmlFor` and `id`, and explicitly define `aria-label` when the action isn't fully described by visual text.
